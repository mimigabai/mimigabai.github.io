Introduction

This case study will cover key stages such as market research, ideation, technical implementation, prototyping stages, and manufacturing considerations that went into redesigning an existing product to make it interactive.

Witness the transformation of "Sorry!" into a more interactive and engaging gaming experience, where strategic design choices and user-centric principles converge to revive this beloved classic.

The process of developing the redesigned "Sorry!" board game honed a diverse set of skills, including strategic thinking, user-centric design, market research analysis, rapid prototyping, effective problem-solving, and arduino wiring and programming.

Idea Generation

To start, we came up with three concepts for adding electronic sensors and actuators to the game to make it more interactive.

Concept 1

Add a button to play fun sounds from speaker when pressed, add another speaker for sounds played when pawns arrive home, add an RGB LED that turns on by photosensor when box is closed.

Concept 2

Add LEDs turned on by a sensor each time a pawn arrives home, when all four have arrived servo motor spins sorry logo in center of board, button to change color of RGB LED indicating player turn.

Concept 3

Similar to above, add LEDs turned on by sensor each time a pawn arrives home, when all four have arrived servo motor spins sorry logo in center of board, have a button to play fun sounds from speaker

User testing and feedback

In our lab environment, it was a bit difficult to collect feedback from users. However, we spoke with our classmates to get feedback and ideas during the phases of the development of the product. First, in the idea generation phase we ask our desk-mates which features they felt were the most exciting. THe other lab groups agreed that having the speaker to play fun sounds was important. Move towards Higher Fidelity

In our early prototype, the goal was to "do just enough". Our prototype was very low risk and low cost and allowed us to see that our product had potential. Our goal for our next prototype would be for it to be ready for users to intereact with so that we can conduct testing. This would include consolidating the arduino, breadboard and wiring (it can be placed in a box underneath the board), and making the board printed so that it is cleaner and ready for a user to interact with.

For our user testing phase, it would be important to find out how easy it was for users to learn and understand the features we added to the game. Additionally, it would be helpful to find out how much users value and appreciate the new interactions.

Design for Manufacturability

When manufacturing the product to market, we will need to move towards producing customized modules. This will allow for modules that are thinner and more precise to our needs.

Once it is established which custom components need to be made, they can be produced using screen printing as part of the production for the processor and some of the traces. There will also need to be connections to connect the components to form a module (batteries, actuators, processor need to be connected). This can be done with various flat connectors to help keep things together and flat.
